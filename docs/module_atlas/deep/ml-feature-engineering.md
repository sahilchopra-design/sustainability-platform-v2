## 7 · Methodology Deep Dive

> ⚠️ **Guide↔code mismatch flag.** The MODULE_GUIDES entry describes a real ML pipeline — mutual
> information feature selection, PCA with 85% variance at 50 components, KSG estimator, singular
> value decomposition — and the page's footer text repeats these claims verbatim ("Mutual information
> computed using KSG estimator (k=5 neighbors). PCA uses standardized features (z-score) with singular
> value decomposition"). **None of this is computed.** Every "MI Score", "correlation" value, and PCA
> "variance" number is generated by the seeded PRNG `sr(s) = frac(sin(s+1)×10⁴)` or a fixed decay
> curve — there is no training data, no XGBoost run, no eigendecomposition. Sections below document
> the static reference catalog the page actually renders.

### 7.1 What the module computes

A 20-feature illustrative catalog (of a claimed 948) is generated once:

```js
FEATURE_CATALOG[i] = {
  name: fixed list (env_score, soc_score, …, circular_economy),
  category: Base Score(×8) | Velocity(×3) | Acceleration(×1) | Derived(×8),
  importance:   round(100 − i*4.2 + (sr(i*10)*2−1)*8),        // linear decay + ±8 jitter
  dqAvg:        round(2.5 + (sr(i*10)*2−1)*1.2, 1),            // Math.round ignores 2nd arg — see §7.6
  completeness: round(85 + (sr(i*10)*2−1)*12),                 // 73–97%
  correlation:  round(30 + (sr(i*510)*2−1)*25 + (sr(i*20)*2−1)*15),
}
CORR_MATRIX[i][j] = i===j ? 100 : round((sr((i-j)*8+500)*2−1)*40 + (sr(i*j+200)*2−1)*20)   // -60..60 band
PCA_DATA[i]  = { variance: round(max(0.5, 15*e^(-i*0.08))*100)/100,       // fixed exponential decay
                 cumulative: min(100, round((1-e^(-i*0.04))*100*10)/10) } // fixed exponential CDF
MI_RANKING   = [...FEATURE_CATALOG].sort(desc by importance)
```

`DQ_IMPACT` (5 rows: DQ1–DQ5 vs RMSE/R²/featureCount) and `PIPELINE_STATUS` (4 rows: base/velocity/
acceleration/derived family refresh status) are entirely hard-coded constants, not derived from
`FEATURE_CATALOG` at all.

### 7.2 Parameterisation

| Constant | Value | Provenance |
|---|---|---|
| `importance` decay | `100 − i×4.2` | Arbitrary linear ramp chosen so the top feature scores ≈100 and the 20th scores ≈21 — not derived from any trained model's feature-importance output |
| PCA variance curve | `15×e^{-0.08i}` | Fixed exponential shape chosen to look like a plausible scree plot; the "50 PCs explain 85% variance" and "elbow at PC12" claims in the footer are descriptive text keyed to this curve's shape, not measured from real eigenvalues |
| `DQ_IMPACT` RMSE range | 3.2 (DQ1) → 11.2 (DQ5) | Illustrative; matches PCAF's DQ1–DQ5 tiering *concept* (lower DQ tier ⇒ better data ⇒ lower model error) but the specific RMSE/R² numbers are hand-typed, not measured |
| `sr()` seed multipliers | `i*10, i*510, i*20, (i-j)*8+500, i*j+200` | Arbitrary decorrelation constants, platform-standard pattern |

### 7.3 Calculation walkthrough

1. **Feature Catalog (Tab 0)** — filterable table over the 20-row catalog; `dqAvg` colour thresholds
   (`≤2` green, `≤3` amber, else red) styled after PCAF's 1–5 data-quality score convention.
2. **Correlation Matrix (Tab 1)** — 20×20 heatmap; diagonal fixed at 100, off-diagonal cells are
   independent PRNG draws bounded to roughly ±60 — there is no guarantee of matrix symmetry or
   positive-semi-definiteness (a real correlation matrix must be PSD; this one is not constructed to
   be).
3. **Feature Selection (Tab 2)** — bar chart of `MI_RANKING` (the catalog re-sorted by the same
   `importance` field used in Tab 0 — "MI Score" is just a relabelling, not an independently computed
   mutual-information statistic); a `topN` slider (50–500) is cosmetic — it does not refilter
   `MI_RANKING`, which always shows all 20 rows.
4. **PCA Analysis (Tab 3)** — area+line chart of the fixed exponential decay/CDF curves described
   above, annotated with a hard-coded "85% threshold" reference line and a text callout claiming
   "Elbow at PC12" — a specific claim not derivable from the smooth exponential curve shown (a true
   elbow requires a kink, which `15e^{-0.08i}` does not have).
5. **Feature Store Status (Tab 4)** / **DQ Impact (Tab 5)** — static reference tables, no computation.

### 7.4 Worked example

`FEATURE_CATALOG[0]` ("env_score", Base Score): `importance = round(100 − 0×4.2 + (sr(0)×2−1)×8)`.
`sr(0) = frac(sin(1)×10⁴) = frac(8414.7...) ≈ 0.7096`. So the jitter term is
`(0.7096×2−1)×8 = 0.4192×8 = 3.35`, giving `importance = round(100 + 3.35) = 103` → capped implicitly
by display but not in code (values above 100 are possible for early indices, illustrating the
formula is a display heuristic, not a bounded 0–100 score as a real MI ranking would be).

### 7.5 Companion analytics

- **KPI strip** — "Raw Features: 948", "Selected: {topN}", "PCA 85%: 50 PCs" — all three are labels,
  not values computed from the 20-row catalog actually rendered (there is no 948-row array anywhere
  in the file; 948 = 316×3, a narrative multiplier only).

### 7.6 Data provenance & limitations

- **All feature-importance, correlation, and PCA-variance numbers are synthetic PRNG output** —
  there is no trained XGBoost/gradient-boosted model, no KSG mutual-information estimator, and no
  SVD/eigendecomposition anywhere in this file, despite the page's own footer text asserting both.
- `dqAvg: Math.round(2.5 + jitter, 1)` — `Math.round` takes only one argument in JavaScript; the `1`
  is silently ignored, so `dqAvg` is always an integer despite being rendered with `.toFixed(1)` —
  the intended one-decimal rounding to nearest 0.1 does not occur (a minor but real code defect, not
  a modelling one).
- The `topN` slider on the Feature Selection tab has no effect on the rendered `MI_RANKING` — it is
  visually interactive but functionally inert.
- Correlation matrix values are not constructed to be symmetric or PSD, so no valid covariance/PCA
  computation could actually be derived from them even if one were implemented.

**Framework alignment:** Mutual information (KSG estimator, Kraskov–Stögbauer–Grassberger 2004) and
PCA/SVD are named in the guide as the selection methodology but not implemented — see §8 for the
production specification. PCAF DQ 1–5 tiering convention is reflected only in the `DQ_IMPACT` table's
qualitative shape (higher DQ number ⇒ worse fit), not computed from real data-quality tags.

## 8 · Model Specification

**Status: specification — not yet implemented in code.**

### 8.1 Purpose & scope
Implement the real feature-engineering pipeline the guide and page footer describe: a 948-feature
family (316 base ESG/climate scores + 316 3-month velocity deltas + 316 12-month acceleration deltas)
feeding mutual-information feature selection and PCA dimensionality reduction for the platform's
downstream ML risk-scoring models (`ml-risk-scorer`, `ml-taxonomy-scoring`).

### 8.2 Conceptual approach
Use **KSG (Kraskov–Stögbauer–Grassberger, 2004) k-nearest-neighbour mutual information estimation**
for feature selection (the non-parametric standard for continuous ESG scores, avoiding the
Gaussianity assumption of Pearson correlation) combined with **standardized PCA via SVD** for
dimensionality reduction — the same combination scikit-learn's `mutual_info_regression` +
`sklearn.decomposition.PCA` pipeline implements, and structurally analogous to how Bloomberg's ESG
factor models and MSCI's risk-factor construction reduce hundreds of raw ESG signals to a compact
factor set.

### 8.3 Mathematical specification

```
Velocity_f(t)     = Score_f(t) − Score_f(t−3mo)
Acceleration_f(t) = Velocity_f(t) − Velocity_f(t−12mo)

MI(f; y) = ∫∫ p(f,y) log[ p(f,y) / (p(f)p(y)) ] df dy     (KSG k-NN estimator, k=5)
Rank features by MI(f;y) descending; select top-N (default 200) subject to |ρ(f_i,f_j)| < 0.7 pruning

Z = standardize(X)                        // zero mean, unit variance per feature
Z = U Σ V^T                               // SVD
PC_k = Z V_k  ;  VarianceExplained_k = σ_k² / Σσ_i²
```

| Parameter | Calibration source |
|---|---|
| KSG neighbour count k | 5 (Kraskov et al. 2004 recommended default for moderate sample sizes) |
| MI top-N selection | 200 of 948 (retains ≥95% cumulative MI mass — validate empirically, not assumed) |
| Correlation pruning threshold | 0.7 (standard multicollinearity cutoff, consistent with the page's own descriptive text) |
| PCA variance target | 85% cumulative (aligns with the page's stated threshold, but must be measured post-hoc, not asserted) |

### 8.4 Data requirements
316 base ESG/climate leaf scores already exist in the platform's taxonomy engine (feeding
`ml-taxonomy-scoring`); velocity/acceleration require point-in-time historical snapshots (3-month and
12-month lookback) of those same scores — likely requiring a new time-series snapshot table if one
does not already exist in the platform's schema.

### 8.5 Validation & benchmarking plan
Benchmark computed MI rankings against a simple univariate F-test ranking (sanity check for gross
disagreement); validate PCA variance-explained curve against scikit-learn's `PCA().fit()` on a held-out
sample; confirm the correlation matrix is symmetric and positive-semi-definite before any
downstream use.

### 8.6 Limitations & model risk
MI estimation with k-NN methods is sensitive to sample size and the curse of dimensionality when many
features are near-duplicates (as ESG sub-scores often are); the pipeline should log estimator
variance/confidence intervals alongside point MI estimates, and PCA components should be inspected
for interpretability (loading review) before being used as opaque model inputs, consistent with SR
11-7 model-development documentation expectations already applied elsewhere on this platform (see
`model-governance`).
